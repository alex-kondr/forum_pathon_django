Нужно создать приложение на Django — простой форум с простой cPannel (Django Admin).
Для UI достаточно использовать готовые решения в духе Bootstrap.
Базовый функционал должен быть покрыт Юнит тестами
В качестве БД - SqlLite

• Должен быть файл сценария для запуска среды разработки. Сценарий должен запускать сервер разработки, инициализировать базу данных sqlite, загружать образцы записей базы данных и т. д.
это не для docker, достаточно обычных средств python/django

Приложение должно иметь следующие страницы:

• Панель инструментов (домашняя страница)

• Список форумов

• Просмотр категории / топика

• Панель администратора

Нет необходимости реализовывать сложный макет HTML. Держите все в стандартных формах django. Аутентификация должна быть стандартной аутентификацией django, но расширенной, чтобы разрешить использование электронной почты и имени пользователя в качестве логина.
Нет необходимости использовать систему ролей.
Формы должны содержать защиту от межсайтового скриптинга.
Уровень базы данных должен обрабатываться с помощью django ORM.
Макеты страниц должны использовать расширяемые шаблоны, чтобы иметь одну главную страницу и отдельные блоги. Административный доступ должен быть предопределенной парой логин/пароль.
Панель администратора должна позволять редактировать категории и подкатегории с подходом django по умолчанию. Никаких дополнительных настроек.

Приложение должно иметь несколько категорий для тем верхнего уровня (категорий) типа «спорт», «часы». Каждая категория должна иметь несколько подкатегорий, например «футбол» в «спорте». И последние темы должны быть размещены внутри подкатегорий. Категория должна иметь отношение «один ко многим» к подкатегориям, подкатегория должна иметь отношение «один ко многим» к потоку.

Некоторые категории должны быть доступны только авторизованным пользователям. На скриншоте это категории «hr», «big boss». Другие категории должны быть открыты как для анонимных, так и для зарегистрированных пользователей.
